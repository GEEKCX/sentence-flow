# 一键补全注释功能说明

## 功能概述

在课程目录添加了"一键补全"按钮，可以为当前课程中的所有句子批量添加单词注释（音标、词性、中文释义）。

## 使用方法

1. 点击顶部进度条，打开课程目录
2. 在课程目录的右上角，点击"一键补全"按钮
3. 确认后，系统会自动处理所有句子
4. 等待处理完成，可以看到进度条和状态提示

## 功能特点

### 1. 智能补全策略
- 优先使用内置词典（ECDICT）
- 词典中没有的单词，使用 AI 缓存中的数据
- 已有注释的单词会被保留，不会被覆盖

### 2. 数据持久化
- 补全后的数据自动保存到 `localStorage`
- 刷新页面后数据不会丢失
- AI 生成的注释会同时保存到 AI 缓存，下次可以快速加载

### 3. 进度显示
- 实时显示处理进度（当前句子/总句子数）
- 显示处理状态（正在加载词典、正在处理句子等）
- 完成后显示处理结果

### 4. 性能优化
- 使用内存缓存和 AI 缓存，提高处理速度
- 异步处理，不阻塞界面
- 批量处理，提高效率

## 工作流程

```
用户点击"一键补全"
  ↓
确认处理
  ↓
加载 ECDICT 词典
  ↓
遍历所有句子
  ↓
对每个单词进行注释补全：
  - 检查是否已有注释 → 有则跳过
  - 在 ECDICT 中查找 → 有则使用
  - 在 AI 缓存中查找 → 有则使用
  - 否则保持为空
  ↓
更新句子数据
  ↓
保存到 localStorage
  ↓
完成
```

## 数据存储

### localStorage 键
- `sentence-flow-course-data`: 存储完整的句子数据（包含所有注释）
- `sentence-flow-ai-word-cache`: 存储 AI 生成的单词注释缓存

### 数据格式
```json
{
  "id": 1,
  "sentence": "Hello world.",
  "translation": "你好，世界。",
  "words": [
    {
      "text": "Hello",
      "phonetic": "/həˈloʊ/",
      "pos": "int.",
      "meaning": "你好"
    },
    {
      "text": "world",
      "phonetic": "/wɜːrld/",
      "pos": "n.",
      "meaning": "世界"
    }
  ]
}
```

## 注意事项

1. **首次使用较慢**：第一次加载 ECDICT 词典需要一些时间
2. **网络要求**：如果使用 AI 补全，需要配置 AI API
3. **数据备份**：建议在批量补全前先导出原始数据
4. **词典限制**：ECDICT 词典包含常用词，生僻词可能无法查到
5. **手动编辑**：补全后可以手动编辑单词注释

## 对比其他功能

| 功能 | 一键补全 | 批量添加音标和注释（编辑器） | 手动单个补全 |
|------|---------|----------------------------|--------------|
| 触发位置 | 课程目录 | 编辑器 | 单词面板 |
| 处理范围 | 当前所有句子 | 编辑器中的句子 | 单个单词 |
| AI 支持 | 仅使用缓存 | 支持实时 AI | 支持实时 AI |
| 速度 | 快（使用缓存） | 中等（可能调用 AI） | 慢（单个处理） |
| 适用场景 | 快速批量补全 | 精确控制 | 精确编辑 |
