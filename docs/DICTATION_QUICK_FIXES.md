## 💡 快速故障排除

### 问题：输入时看不到单词

**检查清单：**
- [ ] 确认当前单词激活了吗？（浅紫色背景）
- [ ] 确认字符显示了吗？（紫色背景的格子）
- [ ] 检查控制台有错误吗？

**解决方法：**
```bash
cd sentence-flow
rm -rf node_modules/.vite
npm run dev
```

### 问题：Ctrl + , 没有反应

**检查清单：**
- [ ] 在默写模式中吗？
- [ ] 按住 Ctrl 了吗？
- [ ] 按的是逗号键吗？（英文输入法）
- [ ] 检查控制台有日志吗？

**解决方法：**
1. 确认使用英文输入法
2. 检查是否有其他应用拦截快捷键
3. 尝试按 `Ctrl + Shift + ,` （如果有 Caps Lock）
4. 尝试点击"偷看答案"按钮代替快捷键

### 问题：看不到控制按钮

**检查清单：**
- [ ] 在默写模式中吗？
- [ ] 页面加载完成了吗？
- [ ] 滚动到页面中间了吗？

**解决方法：**
1. 刷新页面
2. 检查屏幕是否太小
3. 尝试缩放页面（Ctrl + +/-）

```javascript
const listeners = window.getEventListeners?.(window);
console.log('Keydown listeners:', listeners?.keydown?.length);

// 预期输出：至少 1 个
```

### 检查 store 状态

如果可以访问 store（需要从应用导出）：
```javascript
// 检查 isPeeking 状态
console.log('isPeeking:', store.getState().isPeeking);

// 检查 typedChars 状态
console.log('typedChars:', store.getState().typedChars);
```

### 手动测试快捷键

在控制台中运行：
```javascript
// 测试 Ctrl + ,
window.addEventListener('keydown', (e) => {
  if (e.ctrlKey && e.key === ',') {
    console.log('✓ Ctrl + , detected');
    console.log('e.ctrlKey:', e.ctrlKey);
    console.log('e.key:', e.key);
    console.log('e.shiftKey:', e.shiftKey);
  }
});

// 然后按 Ctrl + ,
// 应该看到日志输出
```

## 💡 使用提示

### 最佳学习方式

1. **先听后写**
   - 点击"播放语音"按钮
   - 听 1-2 遍
   - 理解句子意思

2. **凭记忆默写**
   - 不频繁偷看
   - 卡住时再偷看
   - 记住错误的单词

3. **检查回顾**
   - 完成后重听语音
   - 检查拼写是否正确
   - 记录需要练习的单词

### 学习建议

- 📖 每天练习 10-15 个句子
- ⏱️ 每次 20-30 分钟
- 📝 记录错误单词
- 🔄 定期复习错题
- 📈 追踪学习进度
